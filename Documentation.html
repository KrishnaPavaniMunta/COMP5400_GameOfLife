Population Dynamics and Pattern Formation in Stochastic Modifications of John Conway’s Game of Life

Access Code on GitHub
John Conway's Game of Life is a cellular automaton created in 1970 that progresses solely based on its initial configuration. The game's rules of birth, death and survival are determined by the conditions of its immediate neighbours. It is often used to simulate social and biological behaviours, illustrating how complex behaviours can emerge over time from a set of simple rules. However, the game is deterministic in nature, and this project aims to introduce non-deterministic rules to the game to incorporate elements of randomness that align more closely with real-life societies.

Objective

This project aims to explore the impact of stochastic or non-deterministic modifications on the classical Game of Life rules, examining how random alterations in cell deaths lead to new population growth patterns and emergent structures.

Functionalities of extensions

  1. Game of Life Original

    This file includes the original implementation of the Game of Life following all the traditional rules as defined by John Conway. The rules of the game are implemented through the Upgrade Grid function. These rules are as follows:

    • If a living cell has zero or one neighbours, it will die due to loneliness.
    • If a living cell has more than four neighbours, it will die due to overcrowding.
    • If a dead cell is surrounded by exactly three neighbors, a birth will occur, and the cell will become alive.

    Features:

    1. Emergent Patterns: Despite its simple rules, the Game of Life can give rise to complex and interesting patterns. These emergent patterns can provide insights into how simple rules can lead to complex behaviours.
    2. Stable Patterns: Configurations that remain unchanged over multiple generations. These stable patterns include still lifes, such as blocks and beehives, where cells remain fixed in position.
    3. Oscillators: Oscillators are patterns that cycle through a set of configurations repeatedly. They can be simple, like blinkers and toads, or more complex, like pentadecathlons and pulsars.
    4. Spaceships: Some patterns move across the grid over time. These are called spaceships or gliders. They maintain their overall shape as they move, often leaving behind a trail of stable patterns or oscillators.
    5. Growth and Decay: Some patterns may grow indefinitely, while others may stabilise or eventually die out.
    6. Interaction of Patterns: Different patterns interact with each other. They can collide, merge, or influence each other's behaviour in unexpected ways.
    7. Self-Replication: Some configurations in the Game of Life can self-replicate, creating copies of themselves as they evolve. These replicators are fascinating examples of emergent complexity.
    8. Edge Effects: Because the grid is finite, patterns near the edges may exhibit different behaviours compared to those in the central region.

  2. Game of Life with Sacrifice Rules

    This file includes the implementation of a modified Game of Life where the alive cells play a pre-game before every generation called "n-die game". In this pre-game, a cell decides if it wnats to to sacrifice itself if it has "n" neighbors. This is done through a random sequence of the live cells which dictates the order in which cells are checked for the presence of n neighbours. Followin this, the traditional rules are applied. The Upgrade Grid function incorporates the sacrifice rules adopted from the referenced paper by M.R.lauer et al., (1995).

    Features:

    1. Number of neighbours required for a cell to sacrifice itself is required as an input, "n".
    2. Variations of population dynamics for different "n" values.
    3. When "n = 2", new stable patterns other than the defined ones in Conway's Game of Life emerge.

  3. Game of Life with Selfish Rules

    This file includes the implementation of the Game of Life with modified rules where a cell can exhibit selfishness based on a given percentage of selfish births. It introduces the concept of vitality which allows cells to survive events that would naturally lead to their death. Although, for every generation they survive such events, their vitality reduces by one. The Upgrade Grid function incorporates the selfish rules as described and adopted from the referenced paper by Mühlenbernd, R. and Schulz, S. (2014)

    The new rules are defined as:

    1. A cell can survive overcrowding by eliminating its neighbours when surrounded by four or more cells.
    2. A cell can survive uderpopulation even if it has no or one neighbor, as long as its vitality factor is greater than or equal to one.
    3. The third rule allows for births in an empty cell with three or four neighbors.
    4. Finally, the fourth rule designates a new cell as selfish based on a given level of selfishness that is a parameter of the population.

    Features:

    1. Percentage of selfish births is required as an input.
    2. Variations of population dynamics for different percentages of sefish births.

  4. Game of Life with Death Probability

    This file includes the implementation of the Game of Life with modifications that introduce the concept of probability is introduced into the event of cell deaths. Additionally, weighted masks are introduced to vary the influence of the neighbours in the total number of neighbours and hence, in the outcome of the cell's state. The Upgrade Grid function upgrades the rules as described and adopted from the paper by Leonid Yaroslavsky .

    The new rules are defined as:

    1. The "Deaths" of "live" cells are made stochastic with a certain probability, Pdeath, which determines the likelihood of a live cell to die.
    2. The number of "live" cells in the 8-neighbourhood of each cell is counted using a weighted summation which is rounded up to determine the total neighbour count.

    Features:

    1. The probability of death and the weighted mask are requires as inputs for this simulation.
    2. Cells expand infinitely for some probabilities and demonstarte self-healing properties
    3. Some masks influence self-controlled growth in cells.

    5. Note: The theoretical concepts of the modifications have been adopted from existing literature and experimentation has been carried as detailed in the accompanying report.

Dependencies

The main dependencies for running the various versions of Conway's Game of Life are:

  • Python: Python is the primary programming language used for implementing the game and its variations. Ensure you have Python installed on your system. You can download it from the official Python website.
  • pygame: Pygame is a cross-platform set of Python modules designed for writing video games. It includes computer graphics and sound libraries. To install pygame, you can use pip, Python's package manager.
  • numpy: Numpy is a fundamental package for scientific computing with Python. It provides support for large, multi-dimensional arrays and matrices, along with a collection of mathematical functions to operate on these arrays. Install numpy using pip.
  • matplotlib: Matplotlib is a comprehensive library for creating static, animated, and interactive visualizations in Python. It is used for plotting the generation vs. alive cells graph. Install matplotlib using pip.
  • csv (optional): The csv module is used for reading and writing CSV files. It is used in one of the implementations to save data to a CSV file.

Installation Instructions

You can install the dependencies using pip, which is the package installer for Python. Open your terminal or command prompt and execute the following commands:

pip install pygame numpy matplotlib

If you choose to run the version that utilizes the csv module, you can install it with:

pip install csv

These commands will install all the necessary dependencies required to run Conway's Game of Life and its extended implementations.

Once the dependencies are installed, you can run each version of the game by executing the corresponding Python script. For example:

python GameOfLife_CustomPlace.py

Ensure that you are in the correct directory where the Python scripts are located when running the commands. With the dependencies installed, you should be able to enjoy exploring the different implementations of Conway's Game of Life!

How to Play

Game of Life (Original)

  • Run GameOfLife_CustomPlace.py.
  • Hold and drag left-click to place live cells, hold and drag right-click to remove cells.
  • Press spacebar to start/stop the simulation.

Selfish Rules

  • Run GameOfLife_SelfishRules.py.
  • Specify the percentage of selfish births when prompted.

Sacrifice Rules

  • Run GameOfLife_SacrificeRules.py.
  • Specify the number of neighbours required fora cell to sacrifice iself when prompted.

Probability of death

  • Run GameOfLife_DeathProbability.py.
  • Specify the probability of cell death and the weighted mask when prompted.
